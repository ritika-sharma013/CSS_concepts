# CSS_concepts

A personal frontend playground for practicing modern CSS concepts like layout, responsive design, positioning, and styling techniques.

## Overview

This repository is my own study space for experimenting with CSS and making practice layouts. It is not intended as a shared project, but anyone can look if they want to see what I've built.

I use this repo to learn and practice:

- **Flexbox** for one-dimensional layout and alignment
- **Grid** for two-dimensional layout, rows, and columns
- **Responsive design** using media queries and breakpoints
- **Positioning** with static, relative, absolute, fixed, and sticky
- **Box model** concepts like margin, padding, border, and sizing
- **Typography and spacing** for readable layouts
- **Visual styling** with colors, shadows, and transitions

## What I'm Learning

### Flexbox
- `display: flex`
- `flex-direction`, `justify-content`, `align-items`
- `flex-wrap`, `flex-grow`, `flex-shrink`, `flex-basis`
- creating navigation bars, card layouts, and centered content

### Grid
- `display: grid`
- defining rows and columns with `grid-template-columns` and `grid-template-rows`
- grid gaps with `gap`
- item placement using `grid-column` and `grid-row`
- responsive grid rearrangement

### Responsive Design
- using `@media` queries to adapt layouts for different screen sizes
- mobile-first breakpoints and scaling content
- combining flexible units like `%`, `rem`, `vw`, and `vh`

### Positioning
- default `position: static`
- `position: relative` for local offsetting
- `position: absolute` for overlay placement
- `position: fixed` for sticky header/footer effects
- `position: sticky` for elements that stick during scroll

### Layout and Design Patterns
- creating headers, footers, cards, and grids
- designing reusable components with consistent spacing
- using CSS variables for themes and maintainability

## Getting Started

1. Open the project folder in your code editor.
2. Add or modify HTML and CSS files for practice.
3. Preview pages in the browser to check layout and responsive behavior.

## Recommended Practices

- Start with mobile-first CSS and add breakpoints as needed.
- Use semantic HTML structure with descriptive class names.
- Keep layout logic in CSS and content structure in HTML.
- Use reusable utility classes for spacing and alignment.
